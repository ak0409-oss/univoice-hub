UniVoice Design System

Backend-to-Figma Handoff Guide

Goal: A clean, secure Hostel Management System for KIIT.

Global UI Components

These elements must appear on almost every page.

1. Flash Messages (Alerts)
Design a notification area at the very top. Needs styles for:
  • Success (Green) - e.g., "Login Successful"
  • Danger/Error (Red) - e.g., "Wrong Password"
  • Info (Blue/Yellow) - e.g., "Logged out"
2. Navigation Bar
Simple row: Logo (Left) and Logout Link (Right).

1. Login Page (Public)

Visual Requirements

  • Clean, centered card layout.
  • Professional University vibe.
  • Two Distinct Sections: Manual Login vs. Google Login.

  • Must be a link <a>, not a form button.
  • Color: Google Red (#DB4437) or Blue.
  • Text: "Sign in with Google" (or KIIT Email).

3. Management Pages (Split View)

Layout Strategy: Split Screen

Left Side: A Data Table showing existing items.
Right Side: A "Create New" Form stuck to the side.

‚ö†Ô∏è FORM FIELD NAMES (Must Match Exact)

Create Hostel Form:

name="name" (Text), name="gender" (Dropdown: 'Boys', 'Girls'), name="total_rooms" (Number).

Create Warden/Mentor Form:

name="email", name="name", name="password", name="hostel_id" (Dropdown - Wardens only).

4. Manage Students (Complex)

UI Flow

  1. Top Bar: A Dropdown to select "Hostel". The page is empty until this is picked.
  2. Main Table: Shows Room No | Name | Mentor.
  3. Actions: Every row needs a "View Profile" button and "Edit" button.
‚ö†Ô∏è Create Student Form Data
  • name="room_number" (Text - e.g. "101-A")
  • name="hostel_id" (Dropdown)
  • name="mentor_id" (Dropdown)

5. Student Deep Profile

Visual Hierarchy

This page needs to look like a "Medical Record" or "Report Card".

CRITICAL: Flagged Section
If the student has abusive history, this section appears at the top. It must be RED and alarming.
Complaint History Table
Columns: Date, Category, Issue, Status (Color coded badge), Mentor/Warden Comments.

6. Student Dashboard (Frontend)

Key Interactions

  1. "File Complaint" Button: Big, primary action button. Opens a Modal or Form.
  2. My History: A list of their past complaints.
‚ö†Ô∏è New Complaint Form
  • Category Dropdown: name="category" (Electrical, Civil, Mess, Internet, Other)
  • Subject Line: name="heading"
  • Details: name="description"

7. Staff Dashboard

Workflow

This is a "Task List". Staff see complaints assigned to them.

Each Complaint Card needs:

  • Student Info (Name, Room).
  • The Issue (Heading, Description).
  • Action Area: Inputs to solve the issue.
‚ö†Ô∏è Action Form Inputs
  • Status Dropdown: name="status" (values: 'resolved', 'rejected')
  • Comment Box: name="comment" (Text area for feedback)
  • Warden Only: A "Flag as Abusive" checkbox/button.
